Kata incontournables

  • Fizz Buzz : il y a également des variantes. L'objectif est de nous orienter vers une approche plus déclarative basée sur la programmation fonctionnelle.
  • Leap Years : pour le TDD, mais attention à l'ordre des tests.
  • String Calculator : TDD et refactoring.
  • Roman Numerals : Modéliser les correspondances de chiffre sous forme d'enum est une étape clé pour aborder ce kata.
  • Tennis : Il existe 3 version. Objectif refactoring. Très bon kata pour initier la code review. Repo
  • Bowling : illustre le concept de design émergent
  • Gilded Rose : la difficulté se situe dans la compréhension du code, il invite à créer un golden master. Bon kata pour apprendre les raccourcis clavier.
  • Tip Service : refactoring. Difficile à tester. Apprendre à s'affranchir des dépendances, et est une bonne introduction au principe d'inversion de dépendances.

Bibliothèque

Les classiques/fondamentaux

Design patterns: Elements of Reusable Object-Oriented Software - Erich Gamma, Richard Helm, Ralph Johnson, John Vlissides La bible des design patterns.

Head First Design Patterns les patterns les plus utiles et fréquents, en partant d'un problème de conception concret. Très pédagogique.

The Pragmatic Programmer: lecture aisée; attitude et posture du développeur; organisation, choix et maîtrise des outils.

Extreme Programming Explained : une des première méthodes agiles.

Les core craft

Test-Driven Development by exemple : initie à l'approche TDD

Clean code : facile à lire, une bonne porte d'entrée au craft.

Refactoring: Improving the Design of Existing Code - Martin Fowler : la pratique du refactoring expliqué progressivement.

The Software Craftsmanship : attitude et valeur du développeur.

The clean coder : centré sur la production de code de qualité. Facile à lire.

Les crafts avancé

Working Effectively with Legacy Code

xUnit Test Patterns Refactoring Test Code

Domain Driver Design : décrit les patterns stratégiques et tactiques en plaçant le métier au coeur des préoccupation.

Growing Object-Oriented Software : pour développeur confirmé. TDD liées à la collaboration entre les classes.

Living Documentation - Cyrille Martaire
